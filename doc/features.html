Desirable Features for Zen 0.1

Desirable Features for Zen in All Web Browsers

  1. a RESTful interface
  2. unobtrusive JavaScript
  3. progressive enhancement
  4. automatic adjustment to display format
  5. ghosting of option items that are inapplicable in a particular context
  6. injection of JavaScript into web pages from any domain using a web proxy that injects Zen into the page
  7. highest quality fonts
  8. seductive graphic design and typographics (see Friedman and Weingart)
  9. the features of a sophisticated "CSS authoring framework" like Compass (see A Better CSS: Less and SASS to get an idea what's so bad about plain old CSS) -- but for the client side -- with similar imports (see http://compass-style.org/reference/compass/ and plugins (see https://github.com/chriseppstein/compass/wiki/compass-plugins)
  10. good debugging facilities in an environment with jsScheme and client-side continuations
  11. a slew of micro-formats and rich snippets
  12. maybe use Caja

Desirable Features for Zen in Subsets of Web Browsers

It should be demonstrated that these demos do not degrade Zen's operation or performance for any supported browser.

  1. use of many HTML5 features
  2. rounded corners
  3. drop shadows
  4. support for configuration persistence via direct access to CouchDB
